# NOOP v8.5.2

A correctness-and-clarity patch: imported WHOOP journal finally correlates in Insights, metric detail screens stop contradicting their own cards, and the HRV settings are grouped where you'd expect. Update from the Releases page, AltStore, or `brew upgrade`.

## Fixed

- **Imported WHOOP journal correlates in Insights (#136).** WHOOP's `journal_entries.csv` keys each entry only by the cycle's onset evening, but NOOP keys the recovery/sleep it correlates against — and its own native journal — by the **wake day**. So every imported entry sat one day early and never matched its outcome, and Insights bucketed the whole imported history into "Without." Imported entries now key to the wake day. Both platforms. *(New imports are correct immediately; to fix already-imported history, remove and re-add your WHOOP import — a re-import cleans up the old rows within the imported date range and touches nothing outside it.)*
- **Fitness Age / Vitality detail no longer says "Not enough history yet" when the card shows a value (#139).** These weekly-ish scores show on the card from a single reading, but tapping through hit the trend-chart's two-point requirement and dead-ended in a note that contradicted the card. A single reading now shows the value with a "trend fills in once a second reading lands" line — for **every** metric, not just Fitness Age (#146 surfaced it on Vitality). Android catch-up to iOS, which already did this.
- **Fitness Age readiness copy no longer promises a trend it isn't showing.** The detail subtitle now tracks how much history exists (what it still needs / your latest reading / the trend).

## Changed

- **HRV settings grouped under Strap (#155).** The HRV window (whole-night vs deep-sleep, from 8.5.1) shipped in the Units section while the Continuous HRV / Overnight-only toggles live under Strap. It's now in Strap, directly below them, with a one-line description of the trade-off. Both platforms.

## Localization

- **Appearance settings are translated (#152).** "Sky behind cards" / "Card transparency" and their captions now localize (de, es, fr, it, pt-PT, zh-Hans, zh-Hant) instead of falling back to English. Thanks to an outside contribution.

## Notes

- The WHOOP journal fix is **non-destructive** — no existing data is deleted on import. A re-import replaces only the journal within the export's own date range (leaving any older history from a wider export untouched), and it does so atomically, so an interrupted import can't drop the range.
